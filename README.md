# Collaboration Demo Project

This project is used to practice Git and GitHub collaboration workflows.

## Main Workflow

- Open an issue
- Create a branch
- Submit a pull request

## Collaboration Rules

1. Create an Issue before starting major work.
2. Use a dedicated Branch for each change.
3. Write clear and focused Commit messages.
4. Link each Pull Request to its related Issue.
5. Review the proposed changes before merging.
6. Merge only after the CI workflow passes.

## Branch Naming

- `feature/` for new functionality
- `docs/` for documentation
- `release/` for version changes
